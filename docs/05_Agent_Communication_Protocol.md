# Personal CIO System
## Agent Communication Protocol V1.0

---

# 1. Purpose

This document defines how agents and modules communicate inside the Personal CIO System.

The goal is to make all agent outputs structured, auditable, machine-readable, human-reviewable, easy to test, easy to store in DuckDB, and easy to convert into Daily CIO Briefs and Order Drafts.

This protocol is designed for MVP development using a local Python monorepo and DuckDB.

---

# 2. Core Principle

Agents should not exchange vague natural language only.

Every agent should output:

```text
structured data first
natural language explanation second
```

The system should follow this flow:

```text
Market Data
    ↓
Analysis Signals
    ↓
Trade Plans
    ↓
Order Drafts
    ↓
CIO Brief
    ↓
Journal
```

Agents may use natural language for explanation, but all decisions must be traceable to structured fields.

---

# 3. MVP Communication Style

For MVP, all modules communicate through:

- Python function calls
- Pydantic models
- JSON-compatible dictionaries
- DuckDB persistence

No message queue is required in MVP.

Future versions may support REST API, event bus, LangGraph workflow, and async task queues.

# 13. Account Role Rules

Trade Planner must respect account role.

## 13.1 Vanguard Roth IRA

Recommended role:

```text
CORE_PASSIVE
```

Policy:

```text
Individual stock policy: EXIT_ONLY
New individual stock buys: DISABLED
Replacement assets: VOO / QQQ
Daily trade plans: DISABLED except exit plans
```

Example:

```json
{
  "account_id": "VANGUARD_ROTH",
  "account_role": "CORE_PASSIVE",
  "individual_stock_policy": "EXIT_ONLY",
  "new_individual_stock_buy_allowed": false,
  "default_replacement": {
    "VOO": 0.7,
    "QQQ": 0.3
  }
}
```

## 13.2 Schwab Roth IRA

Recommended role:

```text
ACTIVE_TRADING
```

Policy:

```text
Individual stock trading: ENABLED
Order draft generation: ENABLED
Trade Planner: ENABLED
```

Example:

```json
{
  "account_id": "SCHWAB_ROTH",
  "account_role": "ACTIVE_TRADING",
  "individual_stock_policy": "ACTIVE_ALLOWED",
  "new_individual_stock_buy_allowed": true,
```

# 20. Confidence and Score Rules

All confidence scores use 0-100 scale.

Recommended interpretation:

```text
0-30: weak
31-60: moderate
61-80: strong
81-100: very strong
```

Trade Planner rules:

```text
confidence < 50:
  No order draft should be generated.

50 <= confidence < 70:
  Watch only or small-size plan.

70 <= confidence < 85:
  Eligible for order draft.

confidence >= 85:
  Eligible for stronger action, still requiring human approval.
```

# 22. Validation Rules

Before a Trade Plan becomes an Order Draft:

- Symbol must be valid.
- Account must be active.
- Account role must allow the action.
- Position size must be within risk limits.
- Confidence must meet minimum threshold.
- Entry price and stop price must be valid if required.
- No blocked event risk should exist unless explicitly overridden.

Before an Order Draft can be submitted:

- User must approve.
- Broker account must be connected.
- Cash/position availability must be checked.
- Order payload must pass broker validation.
- The draft must link back to a valid Trade Plan.

# 23. MVP Orchestration Flow

```text
1. market-data-service generates MARKET_DATA_SNAPSHOT.
2. analysis-engine generates DAILY_INDICATOR and DAILY_SIGNAL.
3. trade-planner consumes DAILY_SIGNAL and generates TRADE_PLAN.
4. broker-gateway-schwab optionally converts plans into ORDER_DRAFT.
5. personal-cio-agent generates DAILY_CIO_BRIEF.
6. investment-journal records all signals, plans, drafts, decisions and outcomes.
```

# 25. Success Criteria

The Agent Communication Protocol is successful if:

- All module outputs are machine-readable.
- Every action can be traced to underlying signals.
- Every order draft can be traced to a trade plan.
- Every trade plan can be traced to signals and risk rules.
- Every user decision is stored.
- The Daily CIO Brief can be generated from structured data.
- The Investment Journal can evaluate outcomes over time.

The system should remain explainable, auditable, and safe.
